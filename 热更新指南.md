# 热更新指南

> 本指南面向不会写代码的维护者。
>
> 天树刷机支持两种热更新方式：
> - **本地热更新**：直接修改手机上 runtime/ 目录的配置文件，保存后自动生效
> - **服务器同步**：在服务器端修改配置，用户手机上点「同步」按钮拉取最新配置

---

## 一、整体架构

```
服务器端 (81.68.84.205:12345)
  ┌─────────────────────────┐
  │  网页后台 (/admin)       │  ← 你在这里编辑配置
  │  ├─ classifier.yaml      │
  │  ├─ choices.yaml         │
  │  ├─ registry.yaml        │
  │  └─ *.py VIP解析器       │
  └────────┬────────────────┘
           │ /api/config/bundle
           ▼
用户手机 (127.0.0.1:8080)
  ┌─────────────────────────┐
  │  网页「版本」页           │  ← 用户点「同步配置」
  │  ⬇️ 拉取配置             │
  │  写入本地 runtime/       │
  │  自动触发热更新          │
  └─────────────────────────┘
```

## 二、服务器端：网页后台管理

### 2.1 打开后台

1. 在浏览器访问 `http://81.68.84.205:12345/admin`
2. 输入管理密码：`123456`
3. 登录后看到配置文件列表

### 2.2 编辑配置文件

1. 左侧文件列表中点击要编辑的文件
2. 右侧编辑器中修改内容
3. 点击「💾 保存」按钮（或按 Ctrl+S）
4. 保存后自动触发热更新，服务器端立即生效

### 2.3 新建配置文件

1. 点击文件列表上方的「+ 新建」按钮
2. 输入文件路径，如 `vip/my_parser.py`
3. 路径必须以 `rules/`、`vip/`、`classes/` 开头
4. 扩展名必须是 `.yaml`、`.yml`、`.py`
5. 点击「创建」按钮

### 2.4 删除配置文件

1. 选中要删除的文件
2. 点击「🗑 删除」按钮
3. 确认删除

### 2.5 刷新热更新

点击右上角「🔄 刷新热更新」按钮，手动清空所有缓存重新加载配置。

## 三、本地端：从服务器同步配置

### 3.1 同步配置

1. 打开网页 `127.0.0.1:8080`
2. 点击底部导航栏的「版本」页
3. 找到「☁️ 服务器配置同步」区域
4. 确认服务器地址正确（默认 `http://81.68.84.205:12345`）
5. 点击「⬇️ 同步配置」按钮
6. 等待同步完成，会显示「✅ 已同步 N/N 个配置文件」

### 3.2 检查是否有更新

1. 在「☁️ 服务器配置同步」区域
2. 点击「🔍 检查更新」按钮
3. 会显示服务器版本、本地版本、是否有新配置

### 3.3 同步后生效

同步完成后，系统会自动触发热更新，新配置立即生效，无需重启服务、无需重新上传脚本。

## 四、本地热更新（不需要服务器）

如果你直接在手机上修改了 `runtime/` 目录下的配置文件：

1. 保存文件后，系统会在下次上传脚本时自动检测文件变更并重新加载
2. 如果想立即生效，点击网页「版本」页的「🔄 刷新配置」按钮

### 4 个配置文件说明

| 文件 | 位置 | 作用 |
|---|---|---|
| `classifier.yaml` | `runtime/rules/classifier.yaml` | 定义脚本分类规则 |
| `choices.yaml` | `runtime/vip/choices.yaml` | 定义交互式脚本的分支选择项 |
| `registry.yaml` | `runtime/vip/registry.yaml` | 定义 VIP 文件名映射 |
| `pipeline.py` | `runtime/classes/{class_id}/pipeline.py` | 定义特定类型脚本的自定义解析逻辑 |

## 五、修改分类规则

### 5.1 什么时候需要改？

- 有一种新的脚本语法特征需要识别
- 某个特征的检测规则不对

### 5.2 配置文件在哪？

服务器后台编辑：`rules/classifier.yaml`
本地直接修改：`runtime/rules/classifier.yaml`

### 5.3 文件格式

```yaml
bat_features:
  - id: interactive           # 特征唯一 ID
    short: interactive        # class_id 中用的缩写
    pattern: '\bset\s+/p\b'  # 正则表达式
    flags: I                  # I=不区分大小写, S=跨行, M=多行

sh_features:
  - id: for_loop
    short: for
    pattern: '\bfor\s+\S+\s+in\b'
    flags: I
```

### 5.4 添加新特征

在 `bat_features` 或 `sh_features` 列表末尾添加：

```yaml
  - id: call_sub              # 唯一 ID
    short: call               # class_id 中的缩写
    pattern: '\bcall\s+:?\w'  # 正则表达式
    flags: I                  # 不区分大小写
```

保存后自动生效。

## 六、修改交互分支选项

### 6.1 配置文件在哪？

服务器后台编辑：`vip/choices.yaml`
本地直接修改：`runtime/vip/choices.yaml`

### 6.2 顺序抉择（针对特定脚本）

```yaml
sequential_choices:
  "抉择线路.bat":
    - title: "选择boot刷写方式"
      desc: "选择刷写 boot_a 还是 boot_b 分区"
      options:
        - value: "boot_choice=2"     # 脚本中的变量值（不要改！）
          label: "刷写 boot_a"        # 用户看到的名字（改这个）
          description: "flash boot_a boot.img"
          step_count: 1
```

### 6.3 已知分支描述（通用）

```yaml
known_choices:
  "mode=1":
    label: "一键全刷"
    description: "刷写所有分区"
    needs_choice: false
    auto_select: "confirm=y"
```

## 八、添加 VIP 专用解析器

### 8.1 创建解析器

在服务器后台新建文件：`vip/my_parser.py`

```python
def parse(content, script_path="", rom_dir=""):
    from core.hydra import HydraParseResult, HydraStepCompat
    from core.hydra.bat_parser.var_types import CodeBlock, HydraStep

    steps = []
    for line in content.splitlines():
        line = line.strip()
        if not line or line.startswith('@') or line.startswith('rem'):
            continue
        if line.startswith('fastboot flash'):
            parts = line.split()
            if len(parts) >= 4:
                steps.append(HydraStepCompat(
                    type='flash', part=parts[2], fileName=parts[3],
                    params='', raw=line, risk='MEDIUM'
                ))

    bs = [HydraStep(command=f"{s.type} {s.part}", subcommand=s.type,
                    partition=s.part, path=s.fileName, risk=s.risk) for s in steps]
    block = CodeBlock("vip", bs, "VIP解析", "MEDIUM")
    return HydraParseResult(steps=steps, total_steps=len(steps),
                           missing_files=[], blocks=[block],
                           script_type="bat", class_id="vip:my_parser")
```

### 8.2 注册映射

编辑 `vip/registry.yaml`：

```yaml
"我的脚本.bat": my_parser
```

### 8.3 让用户同步

在服务器后台保存后，通知用户在手机上点「⬇️ 同步配置」。

## 九、添加子类覆写

在服务器后台新建文件：`classes/{class_id}/pipeline.py`

```python
from core.hydra.pipelines.bat.pipeline import BatPipeline

class Pipeline(BatPipeline):
    class_id = "bat:for+goto+if"

    def __init__(self):
        super().__init__(class_id="bat:for+goto+if")

    def parse(self, content, script_path="", rom_dir="", user_decisions=None):
        result = super().parse(content, script_path=script_path, rom_dir=rom_dir)
        # 自定义修改逻辑
        return result
```

目录名必须和 class_id 完全一致（包括 `:` 和 `+`）。

## 十、日常维护速查表

| 我想做什么 | 在哪里改 | 怎么让用户生效 |
|---|---|---|
| 修改脚本分类规则 | 服务器后台 → `rules/classifier.yaml` | 用户点「同步配置」 |

## 十一、注意事项

1. **服务器后台密码**：默认 `tianshu2024`，通过环境变量 `ADMIN_PASSWORD` 修改
2. **服务器地址**：默认 `http://81.68.84.205:12345`，用户可在网页上修改
3. **安全白名单**：只允许同步 `rules/`、`vip/`、`classes/` 目录下 `.yaml`、`.yml`、`.py` 文件
4. **YAML 格式**：缩进用空格（不能用 Tab），冒号后面要有空格
5. **文件编码**：所有配置文件用 UTF-8 编码
6. **PyYAML**：如果没有安装 PyYAML（`pip install pyyaml`），choices.yaml 和 classifier.yaml 不会加载，建议安装
7. **网络要求**：服务器同步需要手机能访问服务器地址，没网时用本地配置
8. **覆盖逻辑**：同步会覆盖本地同名文件，本地临时修改会被覆盖
