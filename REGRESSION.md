# Flash Tool 回归测试清单

每次发布前，必须在目标环境（后端模式 + WebUSB 模式）执行以下回归测试，确认核心链路未回归。

## 1. 设备检测

- [ ] 后端模式：连接 ADB / Fastboot 设备后点击「检测设备」，状态栏正确显示设备模式、槽位、Bootloader 锁状态。
- [ ] 设备断开后再次检测，状态栏恢复为「未连接设备」，无 JavaScript 报错。
- [ ] 页面刷新后自动恢复设备检测（后端模式下 init.js 自动触发）。

## 2. 线刷解析

- [ ] 选择一个已解压的刷机包，点击「解析脚本」，步骤列表正确展示分区、镜像、命令描述。
- [ ] 解析包含 for 循环 / if exist 的简单 BAT 脚本，步骤展开正确。
- [ ] 解析复杂 BAT / 原生 SH 脚本，前端正确提示「复杂脚本」并给出原因或原脚本预览。
- [ ] 解析完成后「安全摘要」行正确显示设备、Bootloader、步骤数、风险提示。

## 3. 单刷

- [ ] 在「单分区刷写」页面添加一个 Fastboot 刷写任务，选择镜像并执行。
- [ ] 后端模式：进度条正常推进，完成后日志显示「[成功]」。
- [ ] WebUSB 模式：检测 WebUSB Fastboot 设备后，单刷任务正常执行。
- [ ] 删除任务、清空任务列表按钮工作正常。

## 4. 工具箱 VBmeta

- [ ] 进入版本/工具页，VBmeta 镜像地址输入框可见，提示输入绝对路径。
- [ ] 输入有效的 vbmeta 镜像绝对路径（如 `/sdcard/123456/image/vbmeta.img`）后，关闭校验按钮变为可用。
- [ ] 输入非法路径、越界路径或不存在的镜像时，后端返回明确错误提示。
- [ ] 点击「关闭 VBmeta 校验」弹出二次确认，确认后：
  - [ ] 后端模式：使用 `source=path` 调用 `/api/flash`，全局进度条与模块进度条同步更新，成功后提示「关闭校验完成」。
  - [ ] WebUSB 模式：通过 `/api/image/path_blob` 读取镜像并调用 `webusbFastboot.flash`，进度与状态同步。
- [ ] 未输入镜像地址时关闭校验按钮为禁用状态。

## 5. 版本页日志展开

- [ ] 进入「版本」页面，当前版本默认展开，其他版本默认折叠。
- [ ] 点击版本标题可折叠/展开，箭头方向同步变化。
- [ ] 版本号、发布日期正确渲染，无 XSS 风险（内容已转义）。

## 6. 主题切换

- [ ] 首次访问（无 localStorage 主题）根据北京时间自动选择 light/dark。
- [ ] 点击主题切换按钮，light/dark 循环切换，按钮图标同步变化。
- [ ] 刷新页面后主题保持上次选择。
- [ ] 切换主题不影响其他 UI 状态与日志内容。

## 7. WebUSB

- [ ] 切换到 WebUSB 模式，点击连接/检测按钮，浏览器弹出设备选择授权。
- [ ] 授权后状态栏显示「WebUSB · Fastboot 已连」或「ADB 已连」。
- [ ] WebUSB 模式下线刷按钮禁用并提示「WebUSB 模式不支持线刷」。
- [ ] WebUSB 断开后状态恢复，无报错。

## 8. 上传

- [ ] 点击底部悬浮「上传」按钮，上传弹窗正常打开。
- [ ] 选择本地文件后点击提交，进度与结果提示正常，同一文件不会重复上传。
- [ ] 上传过程中「确认上传」按钮禁用，完成后可重新选择文件。
- [ ] 脚本列表中点击任意脚本，只打开一次查看器，点击关闭或遮罩后查看器完全关闭。
- [ ] 关闭弹窗后再次打开，文件列表/状态重置或保持符合预期。
- [ ] 上传完成后相关列表（如刷机包列表、脚本列表）自动刷新。

## 9. 日志导出

- [ ] 在任意页面产生若干日志后，点击「导出日志」按钮。
- [ ] 下载的 `.txt` 文件包含：导出时间、页面名、运行模式、线刷项目、脚本、步骤数、设备信息、槽位、Bootloader 状态、断点信息以及完整运行日志。
- [ ] 日志内容中文正常、无乱码。
- [ ] 日志超过 2000 行后，旧日志自动移除，导出文件大小可控。

## 10. 初始化无报错（v3.9.0 新增）

- [ ] 首次加载或刷新页面，控制台无 `loadVbmetaRomList is not defined` 或类似未定义函数引用报错。
- [ ] `init.js` 中 `Promise.all` 并行加载完成后，日志打印「初始化加载成功」而非「初始化加载失败」。
- [ ] 解压刷机包完成后，不因缺少函数定义导致 UI 阻塞或白屏。

## 11. 线刷参数输入框（v3.9.0 新增）

- [ ] Web Component 模式下，脚本选择行下方显示参数输入框，占满一行宽度。
- [ ] 输入框占位符为「带参数解析默认无，如需请填写」。
- [ ] 空值输入时解析脚本请求不携带 `extra_args` 字段。
- [ ] 非空值输入时解析脚本请求携带 `extra_args` 字段，后端正确接收并展开。
- [ ] 旧版 `index.html` 线刷页面的 `#batExtraArgs` 输入框依然正常工作，未被本次改动破坏。

## 12. 工具箱 VBmeta 文件选择（v3.9.0 新增）

- [ ] `<tools-panel>` 组件内 VBmeta 区域显示「📂 选择」按钮。
- [ ] 点击「选择」按钮，弹出已解压刷机包项目列表供用户选择。
- [ ] 选中项目后自动扫描 `images/` 目录，找到 `vbmeta.img` 并填入绝对路径。
- [ ] 无已解压项目时提示用户先解压刷机包。
- [ ] 用户仍然可以手动输入绝对路径（双模式不冲突）。
- [ ] 输入框中内容变化时，「执行关闭校验」按钮的禁用状态同步更新。
- [ ] 点击「执行关闭校验」正确调用 `/api/flash{partition: 'vbmeta', extra: '--disable-verity --disable-verification', source: 'path', image: ...}`。
