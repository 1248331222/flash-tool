# 新脚本接入流程

> 本指南面向不会写代码的维护者。
>
> 天树刷机支持两种配置修改方式：
> - **服务器后台**：在 `http://ip地址或网址/admin` 编辑配置，用户手机点「同步配置」拉取
> - **本地修改**：直接在手机上修改 `runtime/` 下的文件，保存后自动生效
>
> 按照下面的步骤操作，就能让一个新的刷机脚本被正确识别和执行。

---

## 一、整体流程概览

```
上传脚本 → 系统自动分类 → 查看解析结果 → 是否正确？
                                    ├─ 是 → 完成，可以直接刷机
                                    └─ 否 → 选择修复方式
                                              ├─ 分支选项不对 → 改 choices.yaml（服务器后台或本地）
                                              ├─ 镜像路径不对 → 检查 ROM 目录结构
                                              ├─ 步骤缺失/多余 → 加 VIP 专用解析器（服务器后台或本地）
                                              └─ 完全无法解析 → 加 VIP 专用解析器（服务器后台或本地）
```

---

## 二、第一步：选择脚本

1. 打开天树刷机网页 `127.0.0.1:8080`
2. 选择相应的刷机脚本（`.bat` 或 `.sh` 文件）
3. 系统会自动调用天树引擎解析脚本

---

## 三、第二步：查看解析结果

点解析后，页面会显示解析结果。你需要检查以下几点：

### 3.1 检查脚本类型（class_id）

页面会显示脚本的分类 ID，例如：
- `bat:bare` — 纯 fastboot 命令脚本（最常见）
- `bat:for+nested+if+delayed+dyn+err_check` — 带循环、条件、延迟展开等复杂语法
- `bat:interactive+for_f+goto+if+delayed+dyn` — 交互式脚本（带 `set /p` 用户选择）

### 3.2 检查步骤列表

每个步骤显示格式：`类型 分区 → 镜像路径 参数`

重点检查：
- **镜像路径**是否正确指向了 ROM 包中的镜像文件
- **参数**是否完整（比如 `--disable-verity --disable-verification` 不能丢）
- **步骤顺序**是否和脚本中的顺序一致
- **步骤数量**是否合理（纯刷机脚本通常 10-60 步）

### 3.3 检查交互式脚本的选择项

如果是交互式脚本（class_id 包含 `interactive`），页面会弹出选择框。检查：
- 选择项的**中文标签**是否正确
- 选择项的**描述**是否清晰
- **选项数量**是否和脚本中的分支数量一致

---

## 四、第三步：如果解析结果不正确

### 情况 A：镜像路径不对

**现象**：步骤显示 `boot → boot.img`，但 `boot.img` 不在当前目录，实际在 `images/boot.img`。

**原因**：脚本中的镜像路径是相对路径，系统没有找到正确位置。

**解决**：
1. 确认 ROM 包的目录结构——脚本文件和镜像文件在同一个目录下吗？
2. 如果镜像是 `images\boot.img` 这种格式，系统会自动把 `\` 转成 `/` 并拼接 ROM 目录
3. 如果脚本用了 `%~dp0` 变量（BAT 脚本自身所在目录），系统会自动解析
4. 如果路径仍然不对，检查 ROM 包是否完整解压

### 情况 B：参数丢失（如 `--disable-verity` 没显示）

**现象**：脚本中有 `fastboot --disable-verity --disable-verification flash vbmeta vbmeta.img`，但步骤只显示 `vbmeta → vbmeta.img`，没有参数。

**解决**：
1. 网页上点「版本」→「🔄 刷新配置」确保配置是最新的
2. 如果仍然丢失，说明 executor 没有正确提取全局参数。需要联系开发者修复 executor

### 情况 C：交互式选择项不对

**现象**：交互式脚本弹出的选择框中，选项标签是英文或者不正确。

**解决**：修改 `runtime/vip/choices.yaml` 文件（详见下文「修改交互分支选项」部分）。

### 情况 D：脚本完全无法解析

**现象**：上传脚本后显示「解析结果为空」或步骤数量明显不对。

**解决**：添加 VIP 专用解析器（详见下文「添加 VIP 专用解析器」部分）。

---

## 五、修改交互分支选项

适用场景：交互式脚本的分支选择标签不对，或需要新增一个交互式脚本的选择项。

### 5.1 找到配置文件

文件位置：`runtime/vip/choices.yaml`

### 5.2 修改顺序抉择（针对特定脚本）

如果你的脚本叫「抉择线路.bat」，找到 `sequential_choices` 部分：

```yaml
sequential_choices:
  "抉择线路.bat":
    - title: "选择boot刷写方式"
      desc: "选择刷写 boot_a 还是 boot_b 分区"
      options:
        - value: "boot_choice=2"
          label: "刷写 boot_a"          # ← 改这里：显示给用户看的标签
          description: "flash boot_a boot.img"  # ← 改这里：描述信息
          step_count: 1
```

**字段说明**：
- `title`：这一步选择的标题
- `desc`：这一步选择的描述
- `value`：脚本中对应的变量值（**不要改**，必须和脚本一致）
- `label`：用户看到的名字（**改这个**）
- `description`：详细说明（**改这个**）
- `step_count`：选这个选项后执行几步

### 5.3 修改已知分支描述（针对所有交互式脚本）

找到 `known_choices` 部分：

```yaml
known_choices:
  "mode=1":
    label: "一键全刷"           # ← 改这里
    description: "刷写所有分区"  # ← 改这里
    needs_choice: false         # false=自动选, true=弹框让用户选
    auto_select: "confirm=y"    # needs_choice=false 时自动选的值
```

### 5.4 修改后怎么生效？

**方法一（推荐）：** 保存文件后，直接重新解析脚本，系统自动用新配置。

**方法二：** 网页上点「版本」→「🔄 刷新配置」按钮。

### 5.5 添加新的交互式脚本

如果有一个新的交互式脚本「我的脚本.bat」，在 `sequential_choices` 下添加：

```yaml
sequential_choices:
  "抉择线路.bat":
    # ... 原有内容 ...
  "我的脚本.bat":                 # ← 新增
    - title: "第一步选择"
      desc: "选择刷写方式"
      options:
        - value: "choice=1"
          label: "选项一"
          description: "刷写 boot"
          step_count: 1
        - value: "choice=2"
          label: "选项二"
          description: "刷写 system"
          step_count: 1
```

**注意**：`value` 的值必须和脚本中 `set /p` 变量接收的值完全一致。

## 六、添加 VIP 专用解析器

适用场景：脚本完全无法解析，或解析结果严重错误，标准管线搞不定。

### 6.1 创建 VIP 解析器文件

在 `runtime/vip/` 目录下创建一个 `.py` 文件，比如 `my_script.py`：

```python
# runtime/vip/my_script.py

def parse(content, script_path="", rom_dir=""):
    # 从 core.hydra 导入返回值类型
    from core.hydra import HydraParseResult, HydraStepCompat
    from core.hydra.bat_parser.var_types import CodeBlock, HydraStep

    steps = []

    # 按行扫描脚本内容，手动提取 fastboot 命令
    for line in content.splitlines():
        line = line.strip()
        if not line or line.startswith('@') or line.startswith('rem'):
            continue
        if line.startswith('fastboot flash'):
            parts = line.split()
            # parts = ['fastboot', 'flash', '分区名', '镜像路径']
            if len(parts) >= 4:
                steps.append(HydraStepCompat(
                    type='flash',
                    part=parts[2],         # 分区名
                    fileName=parts[3],     # 镜像路径
                    params='',             # 额外参数
                    raw=line,              # 原始命令行
                    risk='MEDIUM'
                ))
        elif line.startswith('fastboot reboot'):
            steps.append(HydraStepCompat(
                type='reboot',
                part='',
                fileName='',
                params='',
                raw=line,
                risk='LOW'
            ))

    # 组装返回值
    bs = []
    for s in steps:
        bs.append(HydraStep(
            command=f"{s.type} {s.part}",
            subcommand=s.type,
            partition=s.part,
            path=s.fileName,
            risk=s.risk
        ))

    block = CodeBlock("vip", bs, "VIP解析", "MEDIUM")

    return HydraParseResult(
        steps=steps,
        total_steps=len(steps),
        missing_files=[],
        blocks=[block],
        script_type="bat",
        class_id="vip:my_script"
    )
```

### 6.2 注册文件名映射

编辑 `runtime/vip/registry.yaml`：

```yaml
# 添加一行：脚本文件名 → VIP 解析器名
"我的脚本.bat": my_script
```

支持通配符：
```yaml
"wildcard_*.bat": my_script
```

### 6.3 怎么生效？

保存文件后，直接重新上传脚本，系统自动检测文件变更并重新加载。

也可以网页上点「版本」→「🔄 刷新配置」按钮立即刷新。

### 6.4 验证

上传脚本文件，检查：
- 解析结果是否正确
- 步骤数量是否合理
- 镜像路径是否正确

---

## 七、检查清单

每次接入新脚本后，对照以下清单检查：

| 检查项 | 如何验证 | 通过标准 |
|---|---|---|
| 脚本能被识别 | 上传后不报错 | 页面显示步骤列表 |
| class_id 正确 | 查看页面显示的分类 | 与脚本语法特征匹配 |
| 镜像路径正确 | 查看每个 flash 步骤 | 路径指向 ROM 包内实际文件 |
| 参数完整 | 查看 flash 步骤的参数 | `--disable-verity` 等参数不丢 |
| 步骤顺序正确 | 对比脚本原始内容 | 顺序与脚本一致 |
| 交互式选择项正确 | 如果是交互式脚本 | 选项标签为中文，数量正确 |
| 镜像文件存在 | 查看是否有 missing_files | 缺失文件列表为空 |

---

## 八、常见问题

### Q: 上传脚本后显示「解析结果为空」

**A**: 说明脚本的语法特征没有被分类器识别。检查脚本内容是否有 `fastboot` 命令。如果脚本格式特殊，需要添加 VIP 专用解析器。

### Q: 镜像路径显示为相对路径（如 `boot.img`），不是绝对路径

**A**: 这是正常的。系统会在执行刷机时自动拼接 ROM 目录。前端展示的 `imagePath` 字段才是解析后的绝对路径。

### Q: 交互式脚本没有弹出选择框

**A**: 检查以下几点：
1. 脚本的 class_id 是否包含 `interactive`
2. `runtime/vip/choices.yaml` 是否定义了该脚本的选择项
3. 是否在网页「版本」页点了「🔄 刷新配置」使配置生效
4. 如果没有安装 PyYAML，系统会使用内置默认值，确认默认值中包含你的脚本

### Q: 修改了 choices.yaml 但没有生效

**A**: 修改后有两种方式生效：
- **自动**：直接重新上传脚本，系统会自动检测文件变更
- **手动**：网页上点「版本」→「🔄 刷新配置」按钮

### Q: VIP 解析器报错

**A**: 检查以下几点：
1. VIP 解析器文件是否在 `runtime/vip/` 目录下
2. 文件中是否定义了 `parse(content, script_path, rom_dir)` 函数
3. `registry.yaml` 中的文件名是否和上传的文件名完全一致
4. 保存后重新上传脚本，或点「🔄 刷新配置」按钮

### Q: 怎么知道当前配置是否正确加载了？

**A**: 网页上点「版本」→「📊 查看状态」按钮，会显示当前加载的分类规则数量、VIP 映射数量、管线覆写数量。
